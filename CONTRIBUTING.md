# Contributing

Thanks for contributing.

This repository is meant to stay broadly reusable, safe to publish, and easy for AI tools to read.

## Contribution Principles

- prefer generalized workflow guidance over organization-specific code
- keep examples redacted or synthetic unless redistribution is clearly allowed
- optimize for clarity and reproducibility
- preserve the distinction between data logic, template logic, post-processing, and validation

## Good Contributions

- improved acceptance checklists
- clearer AI instructions or skills
- better redaction guidance
- generalized examples of report architecture
- safer publishing practices
- tooling ideas for comparing generated documents with approved references

## Avoid In Public Contributions

- protected report forms
- real patient, customer, or citizen data
- internal file paths or environment details that should not be published
- organization-specific credentials, tokens, or hosts
- large proprietary code dumps unrelated to the generalized method

## Pull Request Guidance

1. Explain what problem the change solves.
2. Explain whether the change improves method, AI-readability, or publishing safety.
3. Keep the repo useful for teams with very different report formats.
